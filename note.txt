Integrating markAsRead controller


Explanation (10 lines):
This function allows users to mark a notification as read.

It pulls the notification ID from the request parameters.

It checks whether the notification exists in the database.

It ensures the logged-in user owns the notification.

If not found or unauthorized, appropriate errors are thrown.

The isRead property is set to true.

The updated notification is saved to the database.

A success message and updated notification are returned.

This improves UX by letting users track read/unread items.

It reinforces user-specific access control for notifications.
