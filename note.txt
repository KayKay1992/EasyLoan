Integrating updateRepayment controller

✅ 10-Line Explanation:
This controller allows admin to update a repayment record.

It fetches the repayment by its unique ID from the database.

If the repayment doesn’t exist, it returns a 404 error.

The admin can update the status, amount paid, due date, method, and evidence.

Only the fields provided in the request will be updated.

The amountPaid is safely converted to a number.

The repayment object is saved back to the database after updates.

The updated record is returned in the response.

This endpoint is useful for verifying or correcting repayment details.

It's designed to ensure admins can maintain accurate repayment records.
